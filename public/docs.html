Waiflo
API Docs

Complete reference for the Waiflo REST API. Manage workflows, design pipelines programmatically, and execute steps with real-time SSE streaming.

Base URL: http://localhost:3000 Auth: Bearer JWT Format: application/json Streaming: text/event-stream
Authentication
All endpoints except /auth/register and /auth/login require a valid JWT token passed as a Bearer header.
# Include in every authenticated request Authorization: Bearer <your_jwt_token>
⏱ Tokens are valid for 30 days. Re-authenticate via POST /api/auth/login to get a fresh token.
Base URL & Errors
All paths below are relative to your server root. Default port is 3000.
http://your-host:3000/api
ℹ Error responses always return a JSON object with an error string field. HTTP status codes follow standard conventions: 400 bad request, 401 unauthorized, 404 not found, 409 conflict.
// Error response shape { "error": "Workflow already exists" }

CRUD on .waiflo.json files. Each user has an isolated directory on the server. The PUT endpoint is used by the visual editor's autosave (Ctrl+S).

Design API GRANULAR
Surgical read/write on workflow structure. These endpoints use recursive deep merge — only the fields you send are touched. Designed for autosave, programmatic pipeline builders, and AI agents that modify workflows without clobbering adjacent fields.
ℹ Deep merge is recursive: sending { "ws_llm": { "temperature": 0.4 } } updates only temperature, leaving provider and model untouched. Arrays are replaced, not merged.
POST /api/workflow/design Create a new workflow ›
Body
Response

dataobjectoptionalInitial workflow data. Defaults to { lang_name, steps: [], workflows: [] }.

409 Workflow already exists.

Execution
Execute a step in isolation with explicit inputs. prompt steps stream output via Server-Sent Events. api steps return a JSON response directly.
POST /api/exec/step Execute a step — SSE or JSON ›
Body
SSE Events
Example
FieldTypeDescription
stepobjectrequiredFull step definition (same shape as stored in steps[])
inputsobjectrequiredKey-value map of input values matching ws_inputs_schema.properties
⚠prompt steps stream via SSE (text/event-stream). api steps return application/json directly. Check ws_type to pick the right handler.

For prompt steps, the response is a Server-Sent Events stream. Three event types:

EventData shapeDescription
token{ text: string }One LLM output token. Accumulate to build the full response.
done{ usage, parsed? }Stream complete. usage contains token counts. parsed contains the JSON-parsed output if the step has an ws_output_schema.
error{ message: string }Execution error. Handle and close the stream.
# SSE stream example event: token data: {"text": "Priority:"} event: token data: {"text": " HIGH"} event: done data: {"usage": {"input_tokens": 142, "output_tokens": 38}, "parsed": {"priority": "HIGH", "category": "billing"}}
# Execute a prompt step — stream the output curl -X POST http://localhost:3000/api/exec/step \ -H "Authorization: Bearer <token>" \ -H "Content-Type: application/json" \ -d '{ "step": { "ws_name": "classify_ticket", "ws_type": "prompt", "ws_llm": { "provider": "anthropic", "model": "claude-sonnet-4-20250514", "temperature": 0 }, "ws_system_prompt": "You are a support triage expert.", "ws_prompt_template": "Classify: {{ticket_body}}\n\nOutput: {{ws_output_schema}}", "ws_output_schema": { "properties": { "priority": { "type": "string" } } } }, "inputs": { "ticket_body": "My invoice is wrong and I need a refund immediately." } }'
